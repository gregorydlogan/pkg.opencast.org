About

The following Linux distributions are currently supported:

Stable

  • CentOS 6.x, 7.x
  • Scientific Linux 6.x, 7.x
  • RedHat Enterprise Linux 6.x, 7.x
  • Fedora (latest stable) (Testing only)

Installation

A detailed description of the installation process can be found in the Opencast Administrator Guides at https://docs.opencast.org/latest/admin/installation/

Sources

If you want to build your own RPM repository you can get the source RPMs from http://repo.virtuos.uos.de/srpms/. Additionally, the RPM Spec files can be found at https://github.com/lkiesow/opencast-rpm-specs.

Troubleshooting

If you encounter any problems during the installation process check if all repositories are installed and enabled properly. Some distributions for example come with a pre-installed but disabled epel repository. The installation of the epel-release package will not fix this.
You can check what repositories are installed and enabled by executing yum/dnf repolist enabled which should give you a list with epel, matterhorn and matterhorn-noarch in it.
If epel was preinstalled but disabled on your system, you can obviously skip the installation of the epel-release package, although an installation will not harm your system. You just need to enable the system by editing the configuration file /etc/yum.repo.d/epel.repo and set the property enabled to 1.
If you get errors that Opencast dependencies are missing, make sure that you are actually running an OS with x86_64 architecture. There is no repository for other architectures.
